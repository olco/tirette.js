Quick presentation

In some user flow, we sometimes want to avoid as much as possible the tiring back and forth between the mouse and the keyboard (or tactile and virtual keyboard). This component therefore allows the user to keep a perception of the main view without having to switch to another device and therefore without total loss of focus. On the engine side, the pull tab (if properly adjusted in terms of accuracy, see below) will require less effort. It will be especially effective for entering "small" numbers because we are limited by the screen resolution. This component is therefore not intended to be used systematically for number entry or value selection, but can still be useful in many cases.

Integration

After integrating the files "tirette.css" and "tirette.js" in the HTML page, you just have to add class='tirette' to the attribute of the desired input or select fields.

<input type="number" class="tirette">

Precision management via the attribute "accuracy".

An "accurarcy" attribute has been implemented to manage the accuracy of the pull tab. The higher this attribute is, the easier it will be for the user to select the chosen value. For example, if the attribute'accuracy' is set to 4, 4 pixels will be used for one unit. If it is at 10, 10 pixels will be used for one unit, and so on.

<input type="number" class="tirette" accuracy="4">

Care must be taken not to set the accuracy too high because it limits the possibilities of maximum values. For example, for an accuracy to 10 pixels on a maximum diagonal of 800px, the maximum value will not exceed 80 (i.e. 800 divided by 10). Depending on the case, this may be appropriate, but it is important to keep in mind the environment in which you want to execute the component.

Example: Number with an accuracy of 4 pixels for a unit

In the example above, an accuracy of 4 pixels per unit was indicated. This means that it takes 4 pixels to increment the value by one unit. This makes it easier to enter data accurately. On the other hand, it reduces the value possibilities on the screen because the component depends on the available resolution.

'min' and 'max' attributes

I recommend the use of input of type "number" because it manages natively the attributes "min" and "max". However, it is possible to specify these attributes for other types of input (e. g. type="text" or "select', see below).

<input type="number" class="tirette" min="10">

The attribute'min', if specified, is used to define the starting value of the entry. The tirette will take this value directly and gradually increase it as it is stretched.

<input type="number" class="tirette" max="50">

The attribute 'max', if specified, is used to define the end value of the entry. The tirette will never exceed this value.

Example : Birthdate

Above is an example of date entry with min and max values. The pull tab cannot exceed the maximum value. The calculation starts from the minimum value (example for the year: 1900).

Example : Number with negative minimum

It is quite possible to specify a'min' and a'max' negative. The only obligation is to have a "min" lower than the "max".

The selection fields

Integration is as simple as for an input field.

<select class="tirette">...</select>

It also works with 'select' fields. In some cases, these fields have a lot of values not always easy to navigate by a series of clicks. Selecting from a list as proposed by default requires both concentration and accuracy. The pull tab offers an alternative that may, in some cases, be more comfortable to use.

Delete an entered value

To do this, simply double-click on the tirette.

Note: this behavior is not very intuitive (because you must have read the documentation to know the behavior).

Compatibility and licensing

The component is developed in standard Javascript (using principles introduced in ES6). As such, it must be able to be used in any type of web projects with different types of frameworks as soon as the component is executed in a browser supporting the different JS functions used.
